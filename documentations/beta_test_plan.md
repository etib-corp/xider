# G-EIP-700_beta_test_plan

## **1. Project context**

XIDER is an Integrated Development Environment (IDE) oriented for XR application development. It enables users to design, customize, and deploy VR/AR experiences using no-code, low-code, and pro-code approaches. It is built to democratize VR/AR development, making it accessible to users with varying levels of technical expertise.

This project is issued from 2 sub projects:

- Evan: an open-source cross-platform graphical API, based on Vulkan, GLFW and OpenXR.
- Guillaume: an open-source cross-platform UI library, XR and desktop software oriented.

## **2. User Roles**

The following roles will be involved in beta testing.

| **Role Name** | **Description** |
| --- | --- |
| Low-code user | An IT enthusiast, non-professional who want to quickly and simply create a XR application. |
| Hard-code user | A developer, who will be able to create a full complete XR project, with more control on what he is doing, and who wants to gain time on set up with cross-compatibility. |

## **3. Feature table**

All of the listed features will be demonstrated during the beta presentation

| **Feature ID** | **User role** | **Feature name** | **Short description** |
| --- | --- | --- | --- |
| F1 | Everyone | Create a cube | Users can create a cube with a simple interaction |
| F2 | Everyone | Create a text | Users scan create a text and write its content |
| F3 | Everyone | Build a project | Users can build their project in the platform needed (desktop or Android app) |
| F4 | Everyone | Simulate scenes | Users can simulate their project in the IDE |
| F5 | Hard-code | Create a script file | Users can create a script file from the interface |
| F6 | Hard-code | Remove a script file | Users can remove a script file from the interface |
| F7 | Hard-code | Load a 3D model object | Users can load a 3D model object from a file |
| F8 | Hard-code | Manage events | Users can deal with interface events |
| F9 | Hard-code | Import an external shader | Users can import their owns shaders in the project |
| F10 | Hard-code | Debugging project | Users can see logs from the project in order to debug |
| F11 | Hard-code | Import audio files | Users can import and use audio files in their project |
